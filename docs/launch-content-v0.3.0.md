## 2. Reddit Post (r/artificial or r/LocalLLaMA)

**Title:** acore v0.3.0 — a single markdown file that gives any LLM persistent identity and memory (zero-question setup, auto-save, works with any AI)

**Body:**

I've been working on a problem that bugs me every day: AI has no memory of who you are.

Every conversation starts from scratch. You re-explain your preferences, your tech stack, your communication style. The AI that helped you debug for three hours yesterday has no idea who you are today.

**acore** is my answer to this. It's a single markdown file called `core.md` that acts as an identity layer for any LLM. You load it as a system prompt, and your AI gets persistent personality, emotional intelligence, and relationship memory.

No databases. No APIs. No accounts. One text file you control.

**What's new in v0.3.0:**

- **Zero-question setup** — `npx @aman_asmuei/acore` auto-detects your name (from git), your platform, and your tech stack. No interactive prompts.
- **Auto-save** — On Claude Code, Cursor, and Windsurf, the AI writes updates directly to your files. The copy-paste workflow is gone.
- **Web wizard** — Non-technical users can generate their core.md from a web page without touching a terminal.
- **Progressive growth** — Your AI suggests adding trust dynamics after 3 sessions, context modes after 5, and memory lifecycle after 10. Complexity is earned, not dumped on you upfront.

**How it works:**

1. Run `npx @aman_asmuei/acore` (or use the web wizard)
2. Get a `core.md` file personalized to you
3. Load it as a system prompt in your AI tool of choice
4. Your AI now knows your name, preferences, communication style, and builds on that over time

The file has 7 sections: Identity, Relationship, Session, Dynamics, Context Modes, Memory Lifecycle, and Instructions. They're ordered by importance so even if your AI truncates the prompt, the critical parts survive.

It works with ChatGPT, Claude, Gemini, Cursor, Windsurf, Claude Code, or literally any LLM that accepts a system prompt. MIT licensed.

There's also a companion project called **amem** — an MCP memory server that automates the knowledge storage side. acore handles identity (human-curated), amem handles knowledge (automated). They work independently or together.

Links:

- GitHub: https://github.com/amanasmuei/acore
- npm: https://www.npmjs.com/package/@aman_asmuei/acore
- amem (companion memory server): https://github.com/amanasmuei/amem

Happy to answer questions or hear feedback.
